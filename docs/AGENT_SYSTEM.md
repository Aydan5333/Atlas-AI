# Atlas Agent System

## Purpose

Atlas should act as the orchestrator. External tools and agents should feed Atlas, not replace it.

The rule:

> Atlas is the boss. Tools are workers. Human approval stays in control.

## Core agents

### Atlas Chief of Staff Agent
Owns priorities, daily command, weekly reviews, task routing, and decision tracking.

### Atlas Architect Agent
Watches emerging technology, studies new tools, protects the roadmap from hype, and turns useful updates into experiments, issues, branches, and architecture notes.

Atlas Architect tracks AI models, agent systems, wearable displays, AR/XR, voice AI, robotics, edge AI, CAD, 3D scanning, automotive engineering tools, creator tools, education technology, grants, sponsor ecosystems, and business automation.

### Survival + Legacy Agent
Organizes emergency checklists, trusted contacts, document maps, legacy letters, family instructions, offline plans, and the Atlas Go Kit.

### TSC Culture Garage Agent
Plans event zones, run-of-show, car selection, safety checklist, sponsor deliverables, QR signups, and recap prompts.

### TSC Pillars Agent
Keeps ART, MUSIC, FASHION, AUTO, TECH / ATLAS, and EDUCATION / ACADEMY visible so TSC does not accidentally become only an auto/audio brand.

### TSC Academy Agent
Builds workshop tracks, curriculum modules, student pathways, mentor lists, skill checklists, and academy outcomes.

### FSAE / Engineering Agent
Tracks CNM-to-UNM preparation, LOBOmotorsports/FSAE contacts, vehicle dynamics topics, fabrication skills, CAD learning, simulation, and engineering storytelling.

### Sponsor Outreach Agent
Tracks B2 Audio, paint brands, energy drink brands, fashion brands, audio companies, sim racing brands, local shops, sponsor decks, outreach status, and follow-ups.

### Contact Pipeline Agent
Tracks people, circles, why they matter, what proof is needed before contact, first message, follow-up date, and relationship stage.

### Content Recap Agent
Creates shot lists, reels, captions, founder videos, sponsor recaps, event photo lists, and proof-of-work packages.

### Grant + Loan Agent
Tracks business assistance programs, deadlines, required documents, eligibility, application status, and follow-up tasks.

### Property Acquisition Agent
Tracks live/work properties, owner financing, zoning calls, lender conversations, inspection needs, and investor documents.

### Security + Privacy Agent
Defines what can be public, what must stay private, how files are backed up, how secrets are handled, and how the offline vault works.

### Atlas Dev Agent
Uses GitHub, Codex, Claude, Replit, and other build tools to turn the roadmap into issues, branches, pull requests, tests, and deployable features.

### Atlas Wearable HUD Agent
Plans the display layer for smart glasses, wearable screens, and future spatial interfaces. The HUD should stay glanceable, safe, and controlled by Atlas Core.

## Tool roles

- ChatGPT / Codex: chief of staff, coding, repo work, architecture, PRs
- Claude: long-form planning, repo reasoning, specs, grant/sponsor drafts
- Perplexity: current research, contacts, grants, sources, events
- Grok: culture radar, content angles, social tone, trend scanning
- Replit: fast prototypes and demos
- GitHub: source of truth for code, docs, issues, and PRs
- n8n / Zapier / Make: automation later, after data rules are clear
- call agents: later, only with consent, scripts, logging, and human approval
- OpenJarvis: lab/reference only until Atlas architecture is stable

## Human approval wall

No agent should autonomously send sensitive emails, make financial decisions, publish family information, make legal commitments, call people, or alter secrets without explicit approval.
